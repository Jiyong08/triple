# Triple 프로젝트 개발 작업 일지 (2026-07-19)

본 문서는 오늘 진행된 Triple 번역/요약 웹 어플리케이션의 편의 기능 개선 및 버그 수정 작업 내역을 정리한 문서입니다.

---

## 🎯 작업 목적
사용자의 번역 및 요약 작업 프로세스를 매끄럽게 지원하고, 데이터 유실 문제를 방지하며, 생성되는 결과물(마크다운 백업본)의 가독성 및 유용성을 극대화하는 것을 목적으로 합니다.

---

## 🛠️ 주요 작업 및 수정 내역

### 1. 개별 탭 파일 로드 시 데이터 유실 버그 수정
- **문제 현상**: 원문, 번역문, 요약문 탭에 개별적으로 파일을 불러올 때, 이미 다른 탭에 입력되어 있던 텍스트가 사라지고 덮어씌워지는 현상이 발생했습니다.
- **해결 내용**: 각 탭에 독립적으로 파일을 불러오더라도 다른 탭의 기존 데이터가 덮어씌워지지 않고 정상적으로 유지되도록 상태 관리 및 데이터 반영 로직을 수정했습니다.

### 2. 'Triple 불러오기' (통합 복원) 기능 구현
- **목적**: '전체 저장' 버튼으로 저장된 하나의 통합 백업 파일을 불러와서 작업 중이던 세 개의 탭 데이터를 그대로 복원할 수 있도록 지원합니다.
- **해결 내용**: 
  - 앱 최우측 상단 헤더 영역에 **📂 Triple 불러오기** 메뉴 버튼을 신설했습니다.
  - 통합 백업 파일에서 `=== SOURCE TEXT ===`, `=== TRANSLATION TEXT ===`, `=== SUMMARY TEXT ===` 마커를 파싱하여 문단을 추출하는 디코더를 구현했습니다.
  - 가장 문단 개수가 많은 탭을 기준으로 삼아 1:1 대조 셀(DOM)을 재구성하고, 인덱스 재인계 및 높이 동기화(`syncBlockHeights`)를 일괄 실행하여 완벽히 이전 작업 상태로 복구되도록 처리했습니다.

### 3. 요약 예외 처리 규칙 추가
- **목적**: 요약 처리가 불필요한 특정 문항에 대해 사용자가 예외를 지정할 수 있도록 합니다.
- **해결 내용**: 요약문 셀에 `'요약 제외'`, `'요약제외'`, `'제외'`라는 텍스트가 작성되어 있는 경우에는 AI 요약 연산을 건너뛰고, 해당하는 번역문의 내용을 그대로 요약 셀로 복사하도록 코드를 수정했습니다.

### 4. Obsidian MD 파일 다운로드 포맷 개선
- **목적**: 마크다운 파일로 결과물을 내보낼 때, 내용 중복을 방지하고 가독성을 높이며 접기/펼치기 서식을 깔끔하게 제공합니다.
- **해결 내용**:
  - **중복 간소화**: 요약문, 번역문, 원문의 내용이 모두 100% 일치할 경우 요약문 1개만 노출되도록 개선했습니다.
  - **접기 상자 적용**: 번역문과 원문 텍스트는 마크다운 문서 내에서 `<details>` 및 `<summary>` 태그로 감싸 기본적으로 숨김 처리(접힌 상태)되도록 하였으며, 클릭 시에만 펼쳐지도록 구성했습니다.
  - **레이아웃 줄바꿈 정렬**: 한 블록 내의 요소들은 줄바꿈 1개(`\n`)로 콤팩트하게 붙이고, 블록과 블록 사이는 줄바꿈 2개(`\n\n`)로 명확히 분할되도록 간격 포맷을 정돈했습니다.
