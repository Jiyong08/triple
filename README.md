# triple - AI 번역 및 요약 에디터 (Development & Operations Guide)

**triple**은 원문(Foreign Text), 번역 결과(Korean Translation), 요약문(Summary)을 **3:4:3 비율**의 16:9 반응형 3분할 화면으로 배치하여, 정밀한 다국어 번역과 핵심 요약 작업을 동시에 수행할 수 있도록 설계된 전문 블록 에디터입니다.

---

## 1. 프로그램 목적
* **정렬형 블록 에디터**: 단락(Cell) 단위로 화면을 분할하고, 번역문과 요약문 쌍둥이 셀의 시작 높이(상하 Y좌표)를 실시간으로 완벽 동기화하여 가독성과 대조 편의성을 극대화합니다.
* **로컬 LLM (Ollama Gemma4) 통합**: 외부 API 키나 네트워크 비용 없이 로컬에서 고성능 인공지능(Gemma4 IT 모델)을 호출해 번역 및 요약 기능을 수행합니다.
* **전문 용어 사전(Glossary) 연동**: 학술적, 고유 명사 번역의 일관성을 위해 매칭 용어집을 자동 가중치 프롬프트로 바인딩합니다.
* **구조화된 개조식 요약**: 요약 시 불필요한 인사말, 키워드, 혹은 '요약:' 접두사 등을 완전히 배제하고 핵심 명사와 명사구 종결(~함, ~음) 형태의 깔끔한 팩트시트(Fact-sheet) 개조식 요약을 제공합니다.

---

## 2. 주요 개발 과정 및 핵심 기능

### ① 1:1 수직 정렬 블록 에디터 (`index.html`)
* **물리적 셀 정렬**: `syncBlockHeights` 함수를 통해 세 패널의 동일 인덱스 셀 높이 중 최댓값을 실시간으로 강제 일치시킵니다.
* **키보드 네비게이션**: 엔터 3번 입력 시 셀 분할, 첫 줄 맨 앞에서 백스페이스 입력 시 상단 셀과 병합하는 네이티브 편집 경험을 구현했습니다.
* **멀티 패널 동조**: 한 패널에서 셀이 나누어지거나 병합/삭제되면, 나머지 쌍둥이 패널들에도 빈 공백 셀이 실시간으로 삽입/삭제되어 인덱스를 1:1로 정확하게 유지합니다.
* **새로고침(초기화) 로직**: `🔄 새로고침` 시 텍스트만 비워지는 문제를 수정하여, 기존의 복잡한 셀 분절 구조를 모두 파괴하고 모든 패널을 완벽히 **단 1개의 빈 입력창**만 남도록 깔끔하게 초기화합니다.
* **스크롤 연동 토글**: 헤더의 `🔗 스크롤 연동` 버튼을 통해 패널 간의 스크롤 동기화 동작을 필요에 따라 실시간으로 켜고 끌 수 있습니다.

### ② Ollama Gemma4 AI 연동 모듈 (`tri_translate.py` & `tri_summary.py`)
* **번역 모듈 (`tri_translate.py`)**:
  * 마크다운 기호(`#`, `*`, `` ` ``, `[]()` 등) 및 레퍼런스 인용구는 번역하지 않고 그대로 보존하도록 마크다운 보호 규칙을 탑재했습니다.
  * `glossary.txt` 용어 사전에 등재된 단어가 있으면 번역 지침 프롬프트로 강제 주입합니다.
* **요약 모듈 (`tri_summary.py` & `tri_summary_gemini.py`)**:
  * 명사형 종결 어미를 지향하는 엄격한 개조식 요약 가이드라인을 제공합니다.
  * 요약할 텍스트에 중요한 예문이나 구체적 예시, 인용구가 있을 경우에는 생략하거나 축약하지 않고 요약문에 그대로 인용 보존합니다.
* **요약 예외 처리 ('요약 제외')**:
  * 요약문 셀에 `'요약 제외'`, `'요약제외'`, `'제외'`라고 기입하면 AI 요약을 실행하지 않고 왼쪽 번역문의 내용을 요약문 셀로 그대로 복사하여 요약이 불필요한 타이틀/목차 작업을 단축시킵니다.

### ③ 안전한 VRAM 및 세션 메모리 최적화
* **GPU 메모리(VRAM) 안전 언로드**: 번역/요약 누적 20회 가동 시마다 `keep_alive: 0`을 전송하여 VRAM 모델을 강제 언로드합니다. 이때 Ollama의 통신 규격을 철저하게 준수하여 `prompt: ""` 및 `stream: False` 등의 필드를 함께 전달함으로써 Ollama 엔진 자체가 죽는 오작동을 원천 차단했습니다.
* **재시도(Retry) 및 타임아웃 방어**: API 타임아웃을 **120초**로 지정하여 모델 재로딩(Cold Start) 지연을 감쇄하고, **최대 3회 재시도 및 점진적 백오프 대기 시간**을 적용해 에러 저항성을 대폭 높였습니다.

### ④ 시스템 네이티브 파일 입출력 및 UI 개편
* **불러오기 & 저장 아이콘화**: 패널 헤더의 버튼 텍스트를 지우고 심플한 폴더(`📂`) 및 저장(`💾`) 아이콘만 남겨 가로 공간을 넓게 확보했습니다.
* **우측 상단 전체 저장**: 세 패널 모두의 내용을 취합하여 하나의 텍스트 문서로 일괄 저장하는 `💾 전체 저장` 버튼을 제공합니다.
* **최상단 컨트롤 배치 최적화**: 사용성 개선을 위해 우측 상단 메뉴를 `[스크롤 연동]` ➡️ `[Triple 불러오기]` ➡️ `[새로고침]` ➡️ `[결과물통합(md)]` ➡️ `[전체 저장]` 순서로 재배열했습니다.
* **Obsidian MD 저장 포맷 개선**:
  * **중복 간소화**: 요약문, 번역문, 원문의 내용이 모두 100% 일치할 경우 요약문 1개만 노출되도록 개선했습니다.
  * **접기 상자 적용**: 번역문과 원문 텍스트는 마크다운 문서 내에서 `<details>` 및 `<summary>` 태그로 감싸 기본적으로 숨김 처리(접힌 상태)되도록 하였습니다.
  * **하이브리드 포맷 감지 (`isLegacyFormat`)**: 과거 파일의 `키워드:`, `요약:` 표지가 있는 텍스트와 최신 버전의 표지 없는 텍스트를 자동으로 감별합니다. 이를 통해 과거 백업 파일을 복원한 상태에서도 데이터 유실 없이 완벽하게 동일성(`isAllSame`)을 판정하고 정제하여 변환합니다.

### ⑤ 2분할 대조 HTML 변환 앱 (`3md2html.py`) 🆕
* **역할**: 요약문, 번역문, 원문이 통합되어 백업된 트리플 마크다운(`.md`) 파일을 깔끔한 **2분할 대조 HTML** 문서로 변환해 주는 오프라인 뷰어 빌더입니다.
* **핵심 기능**:
  * 불필요한 요약문 정보는 고의로 무시 및 배제합니다.
  * 원문을 창 왼쪽에, 번역문을 창 오른쪽에 1:1 대조 방식으로 출력합니다.
  * **수평 동기화 스크롤**: CSS 그리드(Flex) 정렬을 사용하여, 텍스트 줄바꿈 길이에 상관없이 좌우 매칭 단락이 항상 같은 높이에서 시작하고 동시에 스크롤되도록 지원합니다.
  * 단일 HTML 파일로 빌드되어 웹 브라우저만 있으면 오프라인 환경에서도 반응형 다크/라이트 모드 테마 스위칭을 지원하며 바로 읽을 수 있습니다.

## 3. 실행 환경 및 작업 시 유의사항
1. **Ollama 엔진 가동 및 모델 구동**:
   * 로컬 PC에 Ollama 서버가 실행되어 있어야 하며, Gemma4 모델이 다운로드되어 있어야 정상적으로 AI 번역/요약이 동작합니다.
2. **윈도우 콘솔 한글 깨짐 방지**:
   * Windows 환경(CP949)에서의 CLI 한글 인코딩 깨짐을 방지하기 위해 프로그램 실행 시 UTF-8 인코딩 환경 설정이 반드시 필요합니다.
   * **실행 명령어**: `$env:PYTHONUTF8="1"; [Console]::OutputEncoding = [System.Text.Encoding]::UTF8; python app.py`
3. **깃허브 보안 및 제외 필터 (.gitignore)**:
   * 프로젝트 루트에 `.gitignore`가 생성되어 있습니다. 로컬에서 인증용으로 임의 생성한 개인 API Key 파일(`deepl_key.txt`, `gemini_key.txt`) 및 컴파일 캐시 폴더(`__pycache__/`)가 원치 않게 외부 깃허브에 공유되지 않도록 완벽히 방어합니다.
