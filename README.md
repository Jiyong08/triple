# triple - AI 번역 및 요약 에디터 (Development & Operations Guide)

**triple**은 원문(Foreign Text), 번역 결과(Korean Translation), 요약문(Summary)을 **4:4:2 비율**의 16:9 반응형 3분할 화면으로 배치하여, 정밀한 다국어 번역과 핵심 요약 작업을 동시에 수행할 수 있도록 설계된 전문 블록 에디터입니다.

---

## 1. 프로그램 목적
* **정렬형 블록 에디터**: 단락(Cell) 단위로 화면을 분할하고, 번역문과 요약문 쌍둥이 셀의 시작 높이(상하 Y좌표)를 실시간으로 완벽 동기화하여 가독성과 대조 편의성을 극대화합니다.
* **로컬 LLM (Ollama Gemma4) 통합**: 외부 API 키나 네트워크 비용 없이 로컬에서 고성능 인공지능(Gemma4 IT 모델)을 호출해 번역 및 요약 기능을 수행합니다.
* **전문 용어 사전(Glossary) 연동**: 학술적, 고유 명사 번역의 일관성을 위해 매칭 용어집을 자동 가중치 프롬프트로 바인딩합니다.
* **극소 요약 지원**: 요약 시 3~5개의 핵심 키워드(되도록 적게 추출)와 정확히 단 1개의 해라체 문장으로 요약하여, 극도로 짧고 간결한 요약 정보를 제공합니다.

---

## 2. 주요 개발 과정 및 핵심 기능

### ① 1:1 수직 정렬 블록 에디터 (`index.html`)
* **물리적 셀 정렬**: scrollHeight의 최댓값을 공유하여 세 패널의 동일 인덱스 셀 높이를 동적으로 강제 일치시킵니다 (`syncBlockHeights` 함수).
* **키보드 네비게이션**: 엔터 3번 입력 시 셀 분할, 첫 줄 맨 앞에서 백스페이스 입력 시 상단 셀과 병합하는 네이티브 편집 경험을 구현했습니다.
* **멀티 패널 동조**: 한 패널에서 셀이 나누어지거나 병합/삭제되면, 나머지 쌍둥이 패널들에도 빈 공백 셀이 실시간으로 삽입/삭제되어 인덱스를 1:1로 정확하게 유지합니다.
* **화면 스크롤 튐 제어**: 셀 쪼개기/합치기 동작으로 돔(DOM)이 재생성될 때 화면이 튀지 않도록 이전 `scrollTop` 위치를 보존 및 복원하며, 커서 이동 시 `focus({ preventScroll: true })` 옵션을 사용하여 매끄러운 타이핑 반응성을 완성했습니다.

### ② Ollama Gemma4 AI 연동 모듈 (`tri_translate.py` & `tri_summary.py`)
* **번역 모듈 (`tri_translate.py`)**:
  * 마크다운 기호(`#`, `*`, `` ` ``, `[]()` 등) 및 레퍼런스 인용구는 번역하지 않고 그대로 보존하도록 마크다운 보호 규칙을 탑재했습니다.
  * `glossary.txt` 용어 사전에 등재된 단어가 있으면 번역 지침 프롬프트로 강제 주입합니다.
* **요약 모듈 (`tri_summary.py`)**:
  * 핵심 키워드 3~5개를 추출하고 요약문은 단 1개의 해라체 문장으로만 간결하게 출력하도록 프롬프트 지침을 설정했습니다.
  * 요약할 텍스트에 중요한 예문이나 구체적 예시, 인용구가 있을 경우에는 생략하거나 축약하지 않고 요약문에 그대로 인용 보존합니다.
  * 전체 요약 분량이 대상 텍스트 글자 수의 30% 이하 수준을 유지하도록 프롬프트 분량 가이드라인을 부여했습니다.

### ③ 안전한 VRAM 및 세션 메모리 최적화
* **GPU 메모리(VRAM) 안전 언로드**: 번역/요약 누적 20회 가동 시마다 `keep_alive: 0`을 전송하여 VRAM 모델을 강제 언로드합니다. 이때 Ollama의 통신 규격을 철저하게 준수하여 `prompt: ""` 및 `stream: False` 등의 필드를 함께 전달함으로써 Ollama 엔진 자체가 죽는 오작동을 원천 차단했습니다.
* **재시도(Retry) 및 타임아웃 방어**: VRAM이 비워진 직후 다음 요청 시 모델이 디스크에서 다시 로딩(Cold Start)되는 데 걸리는 시간을 감안하여, API 타임아웃을 **120초**로 대폭 늘렸습니다. 또한, 일시적인 로딩 지연이나 커넥션 불안정에 대비해 **최대 3회 재시도(Retry) 및 점진적 백오프 대기 시간**을 적용해 에러 저항성을 대폭 높였습니다.

### ④ 시스템 네이티브 파일 입출력 및 UI 개편
* **불러오기 & 저장 아이콘화**: 패널 헤더의 버튼 텍스트를 지우고 심플한 폴더(`📂`) 및 저장(`💾`) 아이콘만 남겨 가로 공간을 넓게 확보했습니다.
* **우측 상단 전체 저장**: 세 패널 모두에 데이터가 채워져 있을 때, 원문-번역문-요약문 전체 내용과 셀 구분자(`\n\n\n`)를 통일성 있게 취합하여 하나의 텍스트 문서로 일괄 저장하는 `💾 전체 저장` 버튼을 추가했습니다.
* **네이티브 다이얼로그 및 튜플 오류 해결**: pywebview의 시스템 파일 저장창 API를 파이썬 백엔드([app.py](file:///C:/AG/triple/app.py))에 바인딩하되, 반환되는 윈도우 파일 경로가 플랫폼 버전에 따라 **튜플(Tuple)/리스트** 형식으로 반환되는 현상을 자동 감지하고 첫 번째 문자열 경로만 파싱해 내는 **타입 가드**를 마련하여 안전한 파일 쓰기를 완성했습니다.

## 3. 실행 환경 및 작업 시 유의사항
1. **Ollama 엔진 가동 및 모델 구동**:
   * 로컬 PC에 Ollama 서버가 실행되어 있어야 하며, Gemma4 모델(`hf.co/lmstudio-community/gemma-4-E4B-it-GGUF:Q4_K_M`)이 로컬에 다운로드되어 있어야 정상적으로 AI 번역/요약이 동작합니다. (미연결 시에는 고품질 한국어 Mock 데모 데이터로 가동)
2. **윈도우 콘솔 한글 깨짐 방지**:
   * Windows 환경(CP949)에서의 CLI 한글 인코딩 깨짐을 방지하기 위해 프로그램 실행 시 UTF-8 인코딩 환경 설정이 반드시 필요합니다.
   * **실행 명령어**: `$env:PYTHONUTF8="1"; [Console]::OutputEncoding = [System.Text.Encoding]::UTF8; python app.py`
3. **포트 및 프로세스 소켓 관리**:
   * 에디터 창을 닫은 후에 바로 재실행할 때 가끔 이전 백그라운드 pywebview 또는 GUI 연결 소켓이 다 닫히지 않아 실행 충돌이 날 수 있습니다. 이 경우 잠시(3~5초) 기다렸다가 실행하거나, 가동 중인 Python 프로세스를 정리하고 실행해야 안정적으로 구동됩니다.
